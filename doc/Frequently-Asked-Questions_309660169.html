DTK : Frequently Asked Questions


How can I update DTK. Will I lose any data?

You can always download the latest DTK version from this Confluence space. It comes in two flavours, one without terminal and other with terminal for debugg purposes. (dtk.py and dtk_ts.py). You can just replace the old dtk.py file with the new one. All configuration and data will remain untouched and safe as it is saved in .dtk and .dtk_config folders in your Users/<Username> folder in your machine.

I can see in my log panel that sfdx is not updated. Should I update?

DTK is based in SFDX CLI, so in order to be sure all functionalities are working as they should, we recommend to update it as soon as possible.

If you installed sfdx cli directly, then you can run

 sfdx update

in CMD,

If you installed sfdx with NPM you can update with:

npm install --global sfdx-cli



How does API version affects Retrieve/Deploy

Salesforce is continuously updating the API with their releases. Currently we are on 46 while answering this question. Metadata retrieve and deploy format can change on API versions and this changes can be traced in https://developer.salesforce.com/docs/atlas.en-us.220.0.sfdx_cli_reference.meta/sfdx_cli_reference/cli_reference.htm


How can I receive DTK updates?

Please subscribe to changes in the Confluence page containing the download links and will be notified when a new version is uploaded editing the page.


I receive an error when trying to interact in any way with an Salesforce environment on Retrieve, SOQL, Deploy or Script. ERROR running xxx:xxx:xxx:xxx This org appears to have a problem with its OAuth configuration. Reason: invalid_grant - expired access/refresh token.

The token that is created first time you add a sandbox into DTK has been expired. This may be related to a change of configuration in the environment. (For example a Refresh from productive environment)
